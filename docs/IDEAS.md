## 1. リプレイ・共有

データの再利用で価値が出る系。`MatchRecord` (`src/match/records.ts`) はすでにサーバー保存と同形のスキーマで保存しているので、フロントだけで完結する案も多い。

| | 項目 | 見積 | メモ |
|---|---|---|---|
| 💡 | **マッチリプレイ再生** | M | 既保存の `playerMoves / aiMoves / seed` で完全再現可能。`MatchPanel` の保存リストから「再生」ボタン → 1 手ずつ進む / 自動再生 / スクラブ。プレイヤー盤面と ama 盤面を並べると教材性が高い |
| 💡 | **共有 URL** | S | `seed + moves[]` を base64url で `/replay#xxxx` に固める。Twitter / Discord に貼るだけで他人が同じ譜面を観られる |
| 💡 | **盤面共有 URL / QR** | S | 編集モードで作った盤面 + NEXT 列を URL に固められる。「この詰みやってみて」がワンタップ送信 |
| 💡 | **puyop.com URL 互換** | M | ぷよコミュ標準フォーマットへの書き出し / 取り込み。検索性が一気に上がる |
| 💡 | **連鎖アニメの GIF / 動画書き出し** | L | `MediaRecorder` で canvas を録画 |

## ROI 観点で次に着手するならの推奨ショートリスト

過去のブレストの締めくくりで挙げた 3 つを引き続き推す:

1. **マッチリプレイ再生 + 共有 URL** — 既に保存済みデータを生かせて、教材性・拡散性ともに高い
2. **デイリーシード + 統計画面** — 毎日触る理由を作りつつ、過去のレコードに価値を生やす
3. **おじゃまサバイバル / タワー** — 既存の機能(おじゃま対応 / preset 切替)をフル活用してゲームの幅を広げる
