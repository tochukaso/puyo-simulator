## チューニング定数

- `SEARCH_DEPTH = 4`, `BEAM_WIDTH = 10`(`src/ai/heuristic/search.ts`)
- 小連鎖(< 3)ダンプ係数 `0.1`
- 評価重み: chainPotential 5.0 / heightBalance 0.5 / danger 5.0 / connection 0.5
- アニメ: LOCK_PAUSE 200ms / HIGHLIGHT 400ms / POP 150ms / GRAVITY 300ms
- history 最大 100 手
- AI 推論は Web Worker、依存は stable なフィールド等のみ(入力で再計算しない)

## 仕様の勘所(忘れやすい点)

- 盤面は内部 13 行、row 0 は「天井段」で半透明表示
- SPAWN_COL は 0-index の `2`(= 左から 3 列目)
- ゲームオーバーは SPAWN_COL の SPAWN_AXIS_ROW に canPlace できない時
- 軸 R / 子 B などは文字列 `Color = 'R' | 'B' | 'Y' | 'P'`
- 軸-子相対位置の offsets(`rotation -> [dr, dc]`):
  - 0: [-1, 0]  上
  - 1: [0, +1] 右
  - 2: [+1, 0] 下
  - 3: [0, -1] 左
- `enumerateLegalMoves` は 6列 × 4回転 から壁外を除外して最大 22 手、さらに reachability で絞る
- `commitMove` は `canPlace` を自前でチェックしない(呼び出し側で reachability 済み)
- RNG は mulberry32 シード、`rngSeed` を state に持って逐次更新

## 既知の課題 / 次に触るかもしれない場所

- ちぎり時の軸/子の描画は landing 後の最終位置のみ(途中フレームなし)。必要なら Board のアニメをさらに多段化。
- AI は深さ 4 でも 3 連鎖超えは運ゲー寄り。さらに強くするなら:
  - 連鎖尾の形状評価(GTR/階段/折り返し)
  - `chainPotential` を「置いた後に最大何色で何段詰めるか」に拡張
  - または Phase 5: PyTorch DQN → TF.js エクスポート
- アイコン類は Vite テンプレの仮物。PWA デプロイ前に差し替え要。
- E2E は Playwright 最小のみ。スマホ操作(swipe/long-press)の自動テストは未。
- a11y(キーボードのみ操作、aria)まだ軽め。
