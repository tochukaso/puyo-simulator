### 詰まりどころ

- onnx2tf の依存(`onnx_graphsurgeon`, `psutil`, `ai-edge-litert`)が requirements 未網羅 → 手動追加
- tensorflowjs が `pkg_resources` を要求 → `setuptools<81` にダウングレード
- onnx2tf が PyTorch NHWC→NCHW 内部 permute を見て、TF 側の入力軸を意味不明に再配置(`[B, 6, 7, 13]` になる)。`_NCHWExport` ラッパで NCHW 直入力として再 export してクリーンな `[B, 13, 6, 7]` に修正
- onnx 1.19 が ml_dtypes 新版を要求 → `onnx<1.19` に固定
- Node 24 が `util.isNullOrUndefined` を削除 → tfjs-node が起動時失敗 → polyfill 追加

## 考察

- Top-1 agreement が低いのは、ヒューリスティックの「1 手目だけ厳密に正解」が取れていないため。ただし **プレイ全体のスコア期待値では明確に ML が上**。模倣学習が教師の判断を平滑化した結果、探索的な手を選ぶようになり、たまたま連鎖が乗りやすくなった可能性が高い
- パリティ目標(同等)を達成して余りある結果。5a の Done 条件は満たした
- 15% の top-1 一致で ML が勝つ = 「唯一の正解」ではなく「同程度に良い手が複数」ある局面が多い。5b(MCTS)で価値ネットで順位付けすればさらに伸びる余地あり
