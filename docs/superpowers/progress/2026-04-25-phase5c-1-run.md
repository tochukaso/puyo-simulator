## 達成度の評価

ama vs ml-ama-v1 で B/A ratio 145x = 蒸留モデルは ama の 0.7% という結果は、5c-1 spec の「ama の 80%」目標を**大きく下回った**。一方 Heuristic 比 8.27x、5a (ml-v1) 比 1.37x は実用的な改善。

### 想定原因

1. **データ量不足**: 44k samples は目標 100k の半分弱。教師の判断分布が完全に捉えきれず
2. **過学習が早期に発生**: val loss が epoch 5 で底、以降発散。1M params に対し 44k samples は「ちょうど足りない」量
3. **アーキテクチャと教師強度のギャップ**: ama は SIMD bitfield + 形パターン(GTR/Sullen GTR/Fron)+ depth-3 quiescence search の合成で 13 連鎖を出す。1M params の素朴 ResNet では原理的に届かない領域
4. **Top-1 一致率 8%**: モデルが ama の判断を再現できておらず、独自の弱い政策に収束

### 改善案(5c-1 続編 or 5c-2)

A. データ追加(残り 56k 取得 → 計 100k で再学習、過学習軽減期待)
B. 正則化強化(dropout、label smoothing)+ 学習率小さめ
C. モデル拡大(2-5M params、ResNet 12-16 blocks)、ただし更にデータ必要
D. **MCTS を蒸留モデルに乗せる**(5c-2 RL、policy/value head が走査をガイド)
E. ama を WASM 化してブラウザ直積載(蒸留せず ama 本体使う、原案 D)

## 結論

**蒸留路線単独で ama 級は厳しい**。代わりに以下のハイブリッドが現実的:
- 「速い」UX 用の student モデル(現在の policy-ama-v1)
- 「強い」分析モード用に ama 本体(WASM 化 or サーバ側、選択肢 E)

または 5c-2 RL で蒸留モデルを self-improvement する路線(原 5b の縮小版)。

policy-ama-v1 自体は既存の policy-v1 より強い実用品。デフォルト化 OK だが「ama 級」は別の手段が必要、という学びが得られた。
